# Specs

Detailed implementation specifications (plans) that give agents explicit instructions for building features. Specs are one of the 12 leverage points — they're how you transfer your architectural intent to an agent.

## What Goes Here

- Feature specs with architecture, data models, and acceptance criteria
- Migration plans for refactors
- Integration specs for new services or APIs

## Format

Each spec should include:
1. **Overview** — what is being built and why
2. **Architecture** — where it fits in the codebase
3. **Data Models** — types, interfaces, schemas
4. **Implementation Steps** — ordered, actionable steps
5. **Acceptance Criteria** — how to verify it's done
